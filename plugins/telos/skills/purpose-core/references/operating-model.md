# Operating Model

## Flujo recomendado

1. Nombrar el cambio.
2. Declarar los cuatro horizontes.
3. Identificar contexto mínimo.
4. Pedir propuesta al LLM.
5. Revisar contra propósito.
6. Capturar aprendizaje si aporta valor.

## Cuándo pedir más información

Pide más información solo si falta algo imprescindible para:

- formular el propósito con precisión,
- entender una restricción crítica,
- o evitar una propuesta manifiestamente errónea.

No pidas más contexto por ansiedad o completismo.

## Criterio de rechazo

Una propuesta debe cuestionarse o rechazarse si:

- cumple la función pero rompe la arquitectura;
- cumple la función pero viola una restricción;
- cumple la función pero produce un resultado opaco o difícil de mantener;
- o depende de demasiado contexto innecesario.
