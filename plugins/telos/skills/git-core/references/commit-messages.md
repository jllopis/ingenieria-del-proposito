# Commit Messages

## Reglas

- Mensajes claros, concisos y en modo imperativo.
- Un commit por cambio logico.
- Evitar mezclar refactors con features sin relacion.

## Formato recomendado

<tipo>(<scope>): <resumen>

Tipos sugeridos:
- feat
- fix
- docs
- chore
- refactor
- test

Ejemplos:
- feat(auth): add oauth login
- fix(api): handle null user id
- docs(readme): update install steps
